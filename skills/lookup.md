# Поиск по справочникам и статус аккаунта

Поиск по 12 российским справочникам и проверка состояния аккаунта DaData.

## Инструкция

### Справочники (lookup_reference)

Определи, какой справочник нужен пользователю, и используй `lookup_reference` с соответствующим параметром directory:

| Запрос пользователя | directory | Пример |
|---------------------|-----------|--------|
| Вид деятельности, ОКВЭД | `okved2` | "разработка ПО", "62.01" |
| Продукция, услуги, ОКПД | `okpd2` | "программное обеспечение" |
| Муниципальное образование | `oktmo` | "Москва", "45000000" |
| Станция метро | `metro` | "Арбатская", "Невский проспект" |
| Налоговая инспекция | `fns_unit` | "7700", "ИФНС 46" |
| Таможенный орган | `fts_unit` | "Шереметьевская" |
| Суд | `region_court` | "Арбитражный суд Москвы" |
| Валюта | `currency` | "доллар", "USD", "EUR" |
| Товарный знак, МКТУ | `mktu` | "программное обеспечение" |
| Профессия рабочего | `profession` | "токарь", "сварщик" |
| Должность служащего | `position` | "бухгалтер", "инженер" |
| Медицинская должность | `medical_position` | "терапевт", "хирург" |

### Статус аккаунта

- **Проверить баланс и лимиты** → `get_balance` — покажет остаток в рублях и количество запросов за сегодня
- **Проверить актуальность данных** → `get_versions` — покажет когда обновлялись справочники (ФИАС, ЕГРЮЛ, банки и др.)

## Формат ответа

### Для справочников:
```
## Результат поиска: [название справочника]

**Запрос**: [что искали]
**Найдено**: [количество]

1. **[код]** — [название]
2. **[код]** — [название]
...
```

### Для статуса аккаунта:
```
## Статус аккаунта DaData

**Баланс**: [сумма] ₽
**Запросов сегодня**: [кол-во] из [лимит]

**Обновления справочников**:
- ФИАС: [дата]
- ЕГРЮЛ: [дата]
- Банки: [дата]
```
